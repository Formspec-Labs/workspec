## Adapter split ‚Äî workspace refactor (ports + adapters)

Cargo enforces what conventions cannot. Today `Storage`, `AuthProvider`, and the `AppRuntime` wrapper are inverted as traits but every impl sits in the same crate as its consumers ‚Äî nothing prevents a service from importing `SqliteStorage` directly, and adding Postgres / Keycloak / Restate means forking the existing crate or growing `StorageKind` / `AuthKind` enums forever. Cut now, before WS-020 (Postgres) lands as `storage/postgres.rs` next to SQLite. **Three axes of pluggability:** (1) durable storage ‚Äî sqlite today, postgres / json-fs / in-memory wanted; (2) identity ‚Äî local JWT today, Keycloak / Auth0 / OIDC wanted; (3) durable runtime ‚Äî in-memory today, Restate / Temporal wanted (per parent [`CLAUDE.md`](../../CLAUDE.md) "Restate is initial default production adapter").

**Target crate landscape:**

```
crates/wos-server-ports/             # Trait crate. Storage, AuthProvider, RuntimeAdapter, row + auth types. No impl deps.
crates/wos-server-sqlite/            # SqliteStorage + migrations. Depends on ports.
crates/wos-server-auth-jwt/          # JwtAuth. Depends on ports.
crates/wos-server-auth-mock/         # MockAuth (test/dev). Depends on ports.
crates/wos-server-runtime-local/     # AppRuntime over in-memory WosRuntime. Depends on ports + wos-runtime.
crates/wos-server/                   # Composition root. Axum router, services, ServerConfig. Depends on ports + selected adapters via Cargo features.

# Future siblings (drop-in, no edits to existing crates):
crates/wos-server-postgres/          # WS-020.
crates/wos-server-auth-keycloak/     # OIDC adapter.
crates/wos-server-runtime-restate/   # Restate durable runtime adapter.
crates/wos-server-runtime-temporal/  # Temporal alternate.
```

The ratchet is Cargo: a crate that does not list `wos-server-sqlite` in its deps **cannot** import `SqliteStorage`. Conventions stop being load-bearing.
